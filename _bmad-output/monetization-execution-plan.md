## 一、战略定位

### 一句话定位

**"面向长篇作者/编辑的本地 AI 设定集生成器（Series Bible Generator）"**

### 核心逻辑

| 维度 | 决策 | 理由 |
|------|------|------|
| 目标用户 | 网文作者 + 工作室编辑（B 端） | 付费意愿强（为吃饭付费），痛点真实（300万字记不住设定） |
| 商业模式 | 桌面授权买断 + 年度更新续费 | BYOK 模式下边际成本≈0，参考 Aeon Timeline 模式 |
| 竞争策略 | 降维打击 | Plottr/World Anvil 需手动录入；我们是**自动提取**，代际差异 |
| 护城河 | 本地部署 = 数字主权 | 大厂（字节/阅文）推云端 AI → 作者恐惧数据投喂 → 我们的"本地保险箱"价值越高 |

### 不做什么（同样重要）

- **不做 C 端阅读器竞争**：不跟 Kindle/番茄/起点抢读者
- **不承担推理成本**：不做 SaaS 按量计费，用户自带 Ollama 或 API Key
- **不做移动端**：桌面端先打通，移动端是 Phase 3+ 的事
- **不做多人协作**：Studio 版多席位留到 Phase 3

## 二、产品分层

### Free（Reader 版）— 获客漏斗

当前开源版功能，但加入以下软限制：

| 功能 | Free 限制 | 目的 |
|------|-----------|------|
| 书架 | 最多 **5 本** | 体验完整流程（给作者参考同类作品的空间） |
| 章节分析 | 不限 | 让用户看到分析质量 |
| 可视化 | 全部可用（关系图、地图、时间线、百科） | 这是**最佳广告**——视觉冲击力驱动传播 |
| 导出 | **不可用** | 付费钩子 #1 |
| 设定锁定 | **不可用** | 付费钩子 #2 |
| Q&A 问答 | 每本**累计 50 次** | 体验但不够用（不设日限，降低焦虑感） |

### Pro（Writer 版）— ¥299 / $49 买断（含1年更新）

续费更新：¥149/年 / $29/年

| 付费功能 | 用户价值 | 技术复杂度 |
|----------|----------|-----------|
| **Series Bible 导出**（基于调研的 Top 2-3 格式） | 把 TXT 变成可交付资产 | 中 — 需新建导出引擎 |
| **无限书架** | 职业作者多部作品 | 低 — 移除计数限制 |
| **设定锁定/版本化** | 用户编辑不被 AI 覆盖 | 低 — 现有 override 机制产品化 |
| **无限 Q&A** | 写作时随问随查 | 低 — 移除次数限制 |
| **一致性检测**（Phase 2） | 防吃书 | 高 — 需要新的 LLM pipeline |

### Studio（工作室版）— Phase 3，暂不实施

## 三、分阶段执行计划

### Phase 0：市场验证与调研（第 1-3 周）

**目标**：验证需求真实性，并调研作者对导出格式的具体偏好。

**关键原则：在写导出代码之前，先确保我们知道作者真正想要什么格式的设定集。**

#### P0-1：内容武器（第 1 周）

| 任务 | 内容 | 产出 |
|------|------|------|
| P0-1.1 准备案例 | 选 3-5 本不同类型小说（仙侠/玄幻/悬疑/历史/都市），完整跑通分析流程 | 5 套演示数据集 |
| P0-1.2 录制 Demo | 选《水浒传》或《凡人修仙传》，演示：上传 TXT → 自动分章 → 实体预扫描 → LLM 分析 → 关系图 + 世界地图 + 时间线 + 百科 + Q&A | 3 分钟视频 |
| P0-1.3 剪辑文案 | Slogan: "把你的草稿，变成资产。" | 视频 + 文案 |

#### P0-2：导出格式调研（第 1-2 周，与内容制作并行）

**目标用户细分**：
- 网文作者（起点/晋江/番茄）— 个人作者 + 工作室枪手
- 传统出版系列小说作者
- 编剧/脚本作者
- IP 改编编辑（漫画/游戏/影视）

**调研方法**：

| 方法 | 样本量 | 渠道 |
|------|--------|------|
| 问卷调研 | 50-100 份 | 龙的天空、知乎、作者微信群 |
| 深度访谈 | 10-15 人 | 5 网文作者 + 2 出版作者 + 2 编剧 + 3 改编编辑 |
| 竞品分析 | 5 款产品 | Plottr, Aeon Timeline, World Anvil, Campfire, Scrivener |

**问卷核心问题预览**：
1. 您目前如何管理小说设定？（纯记忆/笔记/Excel/专业软件）
2. 您使用过哪些写作工具？
3. 如果工具能自动生成设定集，您最需要哪些格式？（Markdown/Word/PDF/Excel/JSON/XMind/其他）
4. 设定集中哪些内容最有价值？（人物小传/关系图/时间线/地点层级/势力图/概念百科）
5. 您会把设定集分享给谁？（编辑/工作室/改编方/不分享）
6. 导出功能的心理价位？

**深度访谈大纲（30 分钟）**：
- 当前工作流与工具链（5分钟）
- "吃书"与设定管理痛点场景（10分钟）
- 理想设定集格式与内容（10分钟）
- 付费意愿与价格敏感度（5分钟）

**预期输出**：
- 导出格式优先级矩阵（用户需要 × 技术可行）
- 确定 Phase 1 优先实现的 2-3 种格式
- 导出内容模块化配置需求

#### P0-3：落地页与预售（第 2-3 周）

| 任务 | 内容 | 产出 |
|------|------|------|
| P0-3.1 落地页 | 单页网站：Slogan + Demo 视频 + 功能截图 + 早鸟预售入口 | 静态网页 |
| P0-3.2 预售配置 | "¥249 早鸟价（正式版 ¥299）"，微信支付/支付宝 → 手动发 Key | 预售链接 |
| P0-3.3 渠道投放 | 龙的天空/起点作者论坛/B站/知乎/小红书/抖音视频 | 流量数据 |
| P0-3.4 种子用户 | 主动私信联系 20 位目标用户，发 Demo + 邀请体验 | 20+ 反馈 |

#### P0-4：数据埋点准备（第 3 周）

在 Free 版中加入基础埋点框架（默认开启，用户可关闭）：
- 功能使用频率统计
- 实体卡片交互路径
- 性能指标（渲染时间、响应延迟）

详见「八、数据收集与用户隐私」。

### Phase 1：最小付费版 MVP（第 4-7 周）

**目标**：基于 Phase 0 调研结果，交付 Pro 版的核心付费功能。

#### Epic 1：Series Bible 导出引擎

**这是最核心的付费功能，优先级最高。**

**基于调研结果的格式实现优先级**：

| 格式 | 假设优先级 | 实现策略 | 技术复杂度 |
|------|-----------|----------|-----------|
| **Markdown** | P0 | 优先实现，任何编辑器可读 | 低 |
| **PDF** | P0 | Markdown → PDF，图表用截图嵌入（MVP 方案） | 中 |
| **Word (.docx)** | P1 | 调研确认需求后实现 | 中 |
| **Excel (.xlsx)** | P1 | 人物/地点表格导出 | 低 |
| **JSON** | P2 | 给其他工具集成用 | 低 |
| **XMind/其他** | P2+ | 后续根据反馈决定 | 中-高 |

**Story 清单（根据调研结果调整）**：

| Story | 描述 | 技术路径 | 调研依赖 |
|-------|------|----------|----------|
| 1.1 Markdown 导出 | 一键生成结构化 Markdown 文件，包含：人物小传、关系网络（文本描述）、地点层级、时间线、势力 | 新建 `export_bible_service.py`，聚合现有 `entity_aggregator` + `visualization_service` 数据 | 无 |
| 1.2 PDF 导出（MVP） | Markdown → PDF 渲染，图表使用**前端截图**嵌入，而非后端渲染 | 后端：`weasyprint` 或 `markdown2` + `pdfkit`；前端：Canvas/SVG → PNG 截图 API | 无 |
| 1.3 分模块导出 | 允许选择导出哪些模块：人物卷/地理卷/时间线/势力/概念 | 导出配置参数 | 问卷确认模块偏好 |
| 1.4 自定义封面 | 书名 + 作者 + 生成日期的封面页 | 模板渲染 | 无 |
| 1.5 Word 导出（可选） | 如果问卷显示 >40% 用户需要 Word 格式 | `python-docx` 库 | 问卷结果 |

**导出配置面板设计**：
```
Series Bible 导出设置
├── 格式选择：○ Markdown  ○ PDF  ○ Word（如有）
├── 内容模块（勾选）：
│   ☑ 人物志（□简略版 □完整版）
│   ☑ 关系网络（□文本 □图片）
│   ☑ 世界地理
│   ☑ 时间线
│   ☐ 势力变迁
│   ☑ 概念百科
├── 高级选项：
│   ☑ 包含章节引用链接
│   ☑ 包含统计数据
└── [预览] [导出]
```

#### Epic 2：授权系统（最简版）

**不需要在线验证。桌面软件 = 离线 License Key。**

| Story | 描述 | 技术路径 |
|-------|------|----------|
| 2.1 License Key 生成 | 开发者端工具：生成 `XXXX-XXXX-XXXX-XXXX` 格式密钥 | Python 脚本，HMAC 签名 |
| 2.2 License 验证 | 应用启动时读取本地 license 文件，验证签名有效性 | 后端 middleware，不联网 |
| 2.3 功能门控 | Free vs Pro 功能开关：导出、书架数量、Q&A 次数 | 后端 config flag + 前端 UI 提示 |
| 2.4 激活界面 | 设置页面输入 License Key 的 UI | 前端 Settings 页面扩展 |

**关键决策**：

- **不做在线验证**：本地软件 + 隐私承诺 = 离线验证。接受少量盗版（独立开发者的盗版损失远小于正版用户的信任收益）
- **不做试用期倒计时**：Free 版本身就是永久可用的体验版，书架5本限制就够了

#### Epic 3：设定锁定机制

| Story | 描述 | 技术路径 |
|-------|------|----------|
| 3.1 实体锁定标记 | 用户编辑过的实体字段标记为"已锁定" | 扩展 `map_user_overrides` / `world_structure_overrides` 机制 |
| 3.2 重分析保护 | 重新运行分析时，已锁定的设定不被覆盖 | `entity_aggregator` 合并时 check lock flag |
| 3.3 锁定 UI | 实体详情页的"锁定"按钮 + 锁图标 | 前端 EntityCard 扩展 |

#### Epic 4：用户反馈与数据收集系统

| Story | 描述 | 技术路径 |
|-------|------|----------|
| 4.1 应用内反馈入口 | 设置页增加"反馈与建议"按钮 | 前端对话框 + 后端存储 |
| 4.2 使用统计埋点 | 功能使用频率、性能指标（详见第八章） | `telemetry_service.py` |
| 4.3 数据备份/导出 | 一键导出所有分析数据（JSON） | 复用现有 export 机制 |

### Phase 2：核心壁垒功能（第 8-14 周）

**目标**：补齐"一致性检测"——这是竞品无法复制的付费核心。

#### Epic 5：一致性检测（Continuity Check）

这是研究文档评为"生产力工具最硬核付费点"的功能。

**范围调整**：Phase 2 先做简单的属性矛盾检测，时空冲突检测延后。

| Story | 描述 | 技术路径 | Phase |
|-------|------|----------|-------|
| 5.1 设定矛盾检测 | 检测同一实体的属性前后矛盾（如"火灵根"用水术） | LLM 对比不同章节的实体描述，prompt 设计为对比任务 | 2 |
| 5.2 称呼一致性 | 检测别名/称呼在不同章节的一致性 | 基于现有 `AliasResolver` 扩展 | 2 |
| 5.3 检测报告 UI | 以列表形式展示所有检测到的潜在冲突，可跳转原文 | 新页面/面板 | 2 |
| 5.4 时空冲突检测 | 检测角色在不合理时间内跨越大距离 | 对比 `timeline` 事件的时间 vs `map` 距离 | 3 |

#### Epic 6：写作进度模式（无剧透问答）

| Story | 描述 | 技术路径 |
|-------|------|----------|
| 6.1 写作进度标记 | 作者标记"我写到了第 X 章" | 新增 `writing_progress` 字段 |
| 6.2 范围限定问答 | Q&A 仅检索进度范围内的章节 | 修改 chat service 的 `chapter_end` 参数 |
| 6.3 进度前后对比 | "到第 X 章为止，这个角色的设定是什么" | 时间切片查询 |

### Phase 3：增长引擎（第 15 周+）

市场验证通过后的扩展方向，**基于 Phase 0-2 的用户行为数据**排序：

| 方向 | 描述 | 前提条件（数据支撑） |
|------|------|---------------------|
| **类型模板包** | 仙侠/玄幻/悬疑/历史/都市的专用提示词 + 提取规则 | 埋点显示某类型小说分析质量差，且该类型用户占比高 |
| **Docx/JSON 导出** | 补齐导出格式矩阵 | 问卷/反馈中 >30% 用户要求 |
| **Studio 多席位** | 工作室场景 | 有 B 端客户主动询问时 |
| **角色卡导出** | TavernAI V2 格式 | 反馈中多次出现 |
| **VTT 导出** | Foundry VTT module.json | 海外用户占比 >20% |
| **移动端查看** | PWA 或 Electron 打包 | 反馈中"想在手机上看"高频出现 |

| 版本 | 国内价格 | 国际价格 | 包含 |
|------|----------|----------|------|
| Free | 免费 | 免费 | 5 本书、基础可视化、每本累计 50 次 Q&A |

**定价调整说明**：
- 早鸟价从 ¥199 提高到 ¥249，筛选真正有需求的核心用户
- 续费从 ¥99 提高到 ¥149，匹配"类型模板包"等续费专属内容的价值

### 年 1 保守测算

| 指标 | 数值 | 假设 |
|------|------|------|
| 落地页月访问 | 5,000 | 内容营销 + 社区推广 |
| 注册/下载转化率 | 10% | 500 下载/月 |
| Free → Pro 转化率 | 5% | 25 付费/月 |
| 平均客单价 | **¥270** | 早鸟 + 正式价混合 |
| **年收入** | **¥81,000** | 保守估计 |
| 成本 | ≈ ¥0 | BYOK 模式，无服务器成本 |

**盈亏平衡**：几乎没有运营成本（无服务器、无推理费用），第一单就是净利润。关键成本是开发者时间的机会成本。

## 六、验证清单（Go/No-Go 决策点）

### Phase 0 → Phase 1 的门槛

- [ ] 落地页上线，Demo 视频完成
- [ ] **问卷回收 ≥ 50 份，统计显著**
- [ ] **深度访谈 10+ 人，明确 Top 2 导出格式偏好**
- [ ] 20+ 目标用户完成访谈（痛点验证 + 定价敏感度）
- [ ] **关键指标**：收到 ≥ **20** 份早鸟预售订单（¥249 × 20 = ¥4,980）
- [ ] 用户访谈中 ≥ **60%** 明确表示"愿意付费购买导出功能"

**如果未达标**：停下来重新评估定位，可能需要换方向（如转向 B站 Demo 视频引流 → 读者端 DLC 模式）。

### Phase 1 → Phase 2 的门槛

- [ ] Pro 版发布并交付给早鸟用户
- [ ] 正式版累计 **50+** 付费用户
- [ ] NPS ≥ **30**（用户满意度调查）
- [ ] 埋点数据显示核心功能（导出、Q&A、可视化）使用率 > **60%**
- [ ] 用户自发在社区分享/推荐

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| **导出格式与用户需求不匹配** | 中 | 高 | Phase 0 充分调研；MVP 只做 Markdown + PDF；其他格式根据反馈决定 |
| LLM 提取质量不稳定 | 中 | 高 | 持续优化 prompt + fact validation；强化溯源引用（已有） |
| 盗版 License Key 传播 | 中 | 低 | 离线验证接受盗版；Pro 用户享受更新+社区支持的附加价值 |
| 大厂推出类似功能 | 低 | 中 | 大厂一定走云端路线 → 本地隐私是他们无法提供的 |
| 中文 NLP 在特定类型小说效果差 | 中 | 中 | 类型模板（Phase 3）+ 用户编辑/锁定（Phase 1） |
| 用户不愿意配置 Ollama/API Key | 中 | 中 | 写好入门文档；考虑内置轻量模型（如 qwen3:1.7b）|

## 八、数据收集与用户隐私

### 8.1 埋点原则

| 原则 | 说明 |
|------|------|
| **隐私优先** | 所有数据本地存储，不上传服务器；用户可选择关闭 |
| **匿名化** | 不收集小说内容、实体名称等敏感信息 |
| **最小化** | 只收集对改进产品有用的数据 |
| **透明化** | 在设置中明确告知收集哪些数据 |

### 8.2 核心埋点事件

```yaml
# 功能使用频率
event: feature_used
properties:
  feature: string  # "analysis", "reading", "graph", "map", "timeline", "chat", "export"
  duration_ms: int

# 实体卡片交互
event: entity_card_opened
properties:
  entity_type: string  # "person", "location", "item", "org"
  source: string       # "reading_highlight", "graph_node", "map_marker", "chat_link"

# 分析任务
event: analysis_started/completed
properties:
  chapter_count: int
  entities_extracted: int
  duration_seconds: int

# 导出功能
event: export_initiated/completed
properties:
  format: string       # "markdown", "pdf", "docx"
  modules: [string]    # 导出的模块列表

# 性能指标
event: performance_metric
properties:
  metric: string       # "page_load", "graph_render", "chat_first_token"
  duration_ms: int
```

**敏感信息过滤**：小说标题、实体名称、对话内容、文件路径等**绝不收集**。

### 8.3 用户隐私控制面板

```
设置 > 隐私与数据
┌─────────────────────────────────────────┐
│  使用统计                              │
│                                         │
│  ☑ 帮助改进产品（匿名发送使用统计）     │
│                                         │
│  我们收集的信息：                       │
│  • 功能使用频率（如"打开了关系图3次"）  │
│  • 性能指标（如"地图渲染耗时2秒"）      │
│  • 错误报告                             │
│                                         │
│  我们绝不收集：                         │
│  ✗ 小说内容                             │
│  ✗ 人物/地点名称                        │
│  ✗ 对话记录                             │
│  ✗ 文件路径                             │
│                                         │
│  [ 查看已收集的数据摘要 ]               │
│  [ 删除所有本地统计数据 ]                │
└─────────────────────────────────────────┘
```

### 8.4 数据应用场景

| 数据 | 用途 |
|------|------|
| **功能使用热力图** | 识别低频功能，考虑优化或移除 |
| **功能组合分析** | 发现"看了地图的人 80% 也看了时间线"→ 考虑联动功能 |
| **导出格式偏好** | 指导 Pro 版开发优先级 |
| **性能基准** | 发现慢查询/慢渲染，持续优化 |
| **错误聚类** | 优先修复高频错误 |
| **用户旅程分析** | 发现从 Free 到 Pro 的转化路径 |

## 十、决策建议

**立即行动（本周）**：

1. **设计并发布问卷** — 零成本，快速收集定量数据
2. **录制 Demo 视频** — 现有功能足够震撼
3. **准备 3-5 本小说的完整分析数据** — 覆盖不同类型
4. **列出 20 位潜在访谈对象** — 从你的社交网络开始

**不要做的事**：

- **不要**现在就开始写导出引擎代码 —— 等 Phase 0 调研确定格式优先级
- **不要**做 Electron 打包 —— 当前 `localhost:5173` 足够 MVP
- **不要**做国际化 —— 先吃中文网文市场
- **不要**做移动端 —— 桌面端生产力工具不需要
- **不要**做复杂的 PDF 图表自动渲染 —— MVP 用截图方案

**核心信念**：

> 先调研，再开发。如果问卷都填不满 50 份，说明需求可能不存在。
> 如果 20 个人愿意掏 ¥249，这条路就走得更稳。

**关键调整总结**：

| 维度 | 原方案 | 修订后 |
|------|--------|--------|
| Phase 0 时长 | 2 周 | **3 周**（增加调研） |
| 早鸟门槛 | 10 单 ¥199 | **20 单 ¥249**（提高筛选标准） |
| 导出格式 | 直接做 Markdown + PDF | **基于调研选择 Top 2-3** |
| Free 版限制 | 3 本书 + 日限 10 次 Q&A | **5 本书 + 累计 50 次 Q&A** |
| 数据收集 | 未提及 | **加入埋点框架，隐私优先** |
| Phase 2 一致性检测 | 全功能 | **先做属性矛盾，时空冲突延后** |
