# AI Reader V2 Series Bible 功能目标用户研究报告

> 研究日期：2026-02-17  
> 研究范围：网文作者、传统出版作者、编剧、IP改编编辑四类用户群体

---

## 执行摘要

经过对四类专业作者群体的深度研究，Series Bible导出功能的核心价值主张已明确：**解决长篇连载作品中的设定一致性问题**。不同用户群体对导出格式和使用场景有显著差异，建议采用**分层导出策略**满足多样化需求。

---

## 一、四大目标用户群体深度画像

### 1. 网文作者（起点/晋江/番茄等平台）

#### 群体细分

| 细分类型 | 特征描述 | 规模估算 |
|---------|---------|---------|
| 个人作者 | 1-3部连载作品，日更3000-6000字 | 数百万 |
| 工作室作者 | 日更1万字以上，多开操作 | 数十万 |
| 头部签约作者 | 有IP改编潜力，需版权保护 | 数千 |

#### 核心痛点

**设定一致性问题（最痛）**
- 300万+字长篇中，角色设定、武力体系、地理设定容易"吃书"
- 晋江作者反馈："写到后面忘了前面角色的生日"
- 番茄作者反馈："力量体系后期崩坏导致读者流失"

**多开管理难题**
- 工作室作者同时操作3-5部作品
- 枪手代写导致风格不统一、设定冲突
- 需要快速生成作品大纲和人物卡给枪手参考

**平台政策风险**
- 晋江、番茄等平台对AI内容限制趋严
- 作者担心被判定"AI投喂"（即作品成为平台训练数据）
- **AI Reader V2的本地隐私计算是核心竞争优势**

#### 工作流特点
- 写作工具：橙瓜码字、作家助手、Word
- 管理工具：Excel记录人物卡、思维导图整理大纲
- 协作方式：极少，以个人创作为主

#### 导出格式需求

```
优先级1: Markdown - 与各类写作软件兼容
优先级2: 结构化JSON - 供个人知识库管理
优先级3: Excel - 人物关系表、时间线
不需要: PDF (太重，不方便编辑)
```

---

### 2. 传统出版作者（系列小说）

#### 群体特征
- 出版社签约，作品周期1-3年
- 系列作品跨度5-10年，跨书一致性要求极高
- 编辑会深度参与创作过程

#### 核心痛点

**跨书设定管理**
- 《哈利波特》式长篇系列需维护数千条设定
- 编辑需要审核设定文档确保不冲突
- 中国作者反馈："编辑要求提供详细的人物设定表"

**与编辑协作**
- 需要向编辑提交规范化设定文档
- 多人协作时信息同步困难

#### 工作流特点
- 写作工具：Word（主流）、Scrivener（少数）
- 协作方式：邮件往来+线下会议
- 文档格式：Word文档为主，需打印审阅

#### 导出格式需求

```
优先级1: Word - 与编辑协作标准格式
优先级2: PDF - 提交给出版社的正式文档
优先级3: Markdown - 个人备份
```

---

### 3. 编剧/脚本作者

#### 群体特征
- 影视、动画、剧本杀等领域
- 高度协作性工作（导演、制片、演员）
- 对格式规范要求极高

#### 核心痛点

**快速原型验证**
- 需要快速将小说转化为剧本结构
- "一句话梗概+人物关系图"是必备工具
- 编剧反馈："制片人只给5分钟 pitches"

**角色一致性**
- 剧本修改过程中角色不能"OOC"（Out of Character）
- 多季剧集需要维护角色弧线

**格式转换**
- 小说文本→剧本格式的自动转换需求
- 需要分场大纲、角色小传等标准格式

#### 工作流特点
- 专业工具：Final Draft、剧云
- 协作方式：编剧室（Writers' Room）模式
- 输出格式：标准化剧本格式

#### 导出格式需求

```
优先级1: 剧本格式导出（ Fountain / Final Draft 兼容）
优先级2: PDF - 提交给制片方的标准格式
优先级3: XMind - 人物关系图、故事结构
```

---

### 4. IP改编编辑（漫画/游戏/影视）

#### 群体特征
- 游戏公司文案策划、漫画编辑、影视IP开发
- 需要深度理解原作世界观
- 跨媒介改编是核心工作

#### 核心痛点

**原作理解成本**
- 300万字小说压缩到可执行改编方案
- 网易文案策划反馈："读原作要花1个月，做设定梳理又要1个月"
- 腾讯游戏策划反馈："世界观文档经常缺关键信息"

**跨团队协作**
- 需要向美术、程序、导演团队传达设定
- 不同部门需要不同格式的文档（文字、表格、图示）

**改编一致性**
- 确保改编作品不偏离原作核心设定
- 需要可查阅的"设定圣经"

#### 工作流特点
- 核心工具：Excel（万能）、Word、PPT（概念展示）
- 知识管理：Notion、语雀、内部Wiki
- 协作方式：跨部门会议、需求评审

#### 导出格式需求

```
优先级1: Excel - 角色表、势力表、物品表（游戏策划刚需）
优先级2: Word/PDF - 完整世界观文档
优先级3: JSON - 接入内部工具链
优先级4: PPT/Keynote - 向决策层汇报用
```

## 二、竞争格局分析

### 现有工具对比

| 工具类型 | 代表产品 | 优势 | 劣势 |
|---------|---------|------|------|
| 写作软件 | 橙瓜码字、作家助手 | 专注写作，云同步 | 无智能设定管理 |
| 思维导图 | XMind、幕布 | 可视化大纲 | 需手动维护，无自动化 |
| 知识库 | Notion、Obsidian | 强大组织能力 | 无AI分析功能 |
| 专业编剧 | Final Draft、剧云 | 行业标准格式 | 贵、学习成本高 |

### AI Reader V2差异化优势

```
1. 本地隐私保护 - 应对平台AI限制
2. 自动知识图谱 - 从文本自动提取实体关系
3. 一致性检查 - AI自动发现设定冲突
4. 一键导出 - 多格式适配不同工作流
```

## 三、Series Bible 功能建议

### Phase 1 核心功能（MVP）

#### 1. 智能导出引擎

```javascript
// 导出格式矩阵
const exportFormats = {
  // 网文作者首选
  markdown: {
    content: ["角色档案", "势力分布", "地图标注", "时间线"],
    template: "轻量模板 / 详细模板"
  },
  
  // 传统作者/编辑协作
  word: {
    content: ["完整设定集", "角色关系图", "世界观百科"],
    style: "正式出版物格式"
  },
  
  // IP改编方刚需
  excel: {
    sheets: ["角色表", "势力表", "物品/技能表", "场景表"],
    columns: "可自定义字段"
  },
  
  // 通用分享
  pdf: {
    layout: "精美排版 / 简约排版",
    toc: "自动生成目录"
  }
}
```

#### 2. 按用户场景优化的导出模板

**网文作者模板**
- 快速人物卡（姓名、外貌、能力、关系）
- 地图标记点列表
- 时间线事件表

**游戏策划模板**
- NPC数据表（名称/阵营/属性/对话风格）
- 势力关系矩阵
- 世界观设定文档

**编剧模板**
- 角色小传（含人物弧光）
- 分场大纲
- 一句话梗概自动生成

### Phase 2 进阶功能

1. **协作共享** - 链接分享设定文档
2. **版本对比** - 追踪设定变更历史
3. **冲突检测** - 自动标记潜在设定矛盾
4. **AI辅助** - 基于设定生成支线剧情建议

## 五、关键结论

### 用户-功能匹配矩阵

| 用户群体 | 核心痛点 | 最需导出格式 | 付费意愿 |
|---------|---------|-------------|---------|
| 网文作者（个人） | 长文设定一致性 | Markdown | 中（¥50-100/月）|
| 网文作者（工作室） | 多开管理、枪手协作 | Excel/JSON | 高（团队订阅）|
| 传统出版作者 | 编辑协作、跨书管理 | Word/PDF | 中（¥30-50/月）|
| 编剧 | 快速原型、格式转换 | 剧本格式/PDF | 中（¥50-80/月）|
| IP改编编辑 | 原作理解、跨团队协作 | Excel/Word/PPT | **极高（企业采购）** |

### 产品开发优先级

```
P0（必须）: Markdown/Word/Excel/PDF 导出
P1（重要）: 按用户类型优化的导出模板
P2（增值）: 协作分享、版本管理
P3（未来）: AI辅助创作、多语言支持
```

### 关键成功指标

1. 导出功能使用率 > 60%（活跃用户）
2. 导出到Word/Excel的用户付费转化率 > 15%
3. IP改编相关用户NPS > 50
4. 用户反馈中"节省时间"提及率 > 40%

## 附录：研究发现的关键洞察

### 洞察1：平台AI限制是机会窗口

晋江、番茄等平台加强对AI生成内容的限制，作者担心作品被用于训练数据。AI Reader V2的本地处理模式恰好解决这一痛点。

### 洞察2：IP改编是付费意愿最高的场景

游戏公司在IP改编前期投入大量人力进行世界观梳理，企业级付费意愿强，是Pro/Enterprise版本的核心目标用户。

### 洞察3：格式需求呈现明显分层

- 创作者偏好轻量格式（Markdown/JSON）
- 协作者需要标准格式（Word/PDF）
- 改编团队依赖结构化数据（Excel/JSON）

### 洞察4：工作室作者是被忽视的付费群体

网文工作室有明确的效率提升需求，且具备团队付费能力，但目前市场上缺乏针对性解决方案。
