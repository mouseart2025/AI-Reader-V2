# 西游记数据检视标注指南

## 文件清单

| 文件 | 内容 | 预估时间 |
|------|------|---------|
| `xiyouji_hierarchy_tree.txt` | 层级树（缩进格式） | 5 min |
| `xiyouji_characters.json` | 80 个高频人物 | 15 min |
| `xiyouji_aliases.json` | 40 个别名组 | 20 min |
| `xiyouji_locations.json` | 744 个地点（全部） | 30 min |
| `xiyouji_relations.json` | 80 个高频关系对 | 30 min |
| `xiyouji_spatial.json` | 618 个 contains 空间关系 | 15 min（抽查） |
| `xiyouji_spot_check.json` | 5 章随机抽检 | 10 min |

建议按上述顺序检查。

---

## 1. 层级树 `xiyouji_hierarchy_tree.txt`

**怎么看：** 直接打开，看缩进结构是否符合你对西游记世界的认知。

```
天下 (world)
  ├─ 主世界 (tier=continent, mc=0, ch=24)
    ├─ 西牛贺洲 (tier=continent, mc=75, ch=52)
      ├─ 灵山 (tier=region, mc=14, ch=5)
        ├─ 大雷音寺 (tier=site, mc=7, ch=3)
      ├─ 天竺国 (tier=kingdom, mc=9, ch=8)
    ├─ 东胜神洲 (tier=continent, mc=25, ch=10)
      ├─ 傲来国 (tier=kingdom, mc=12, ch=3)
        ├─ 花果山 (tier=region, mc=13, ch=4)
  ├─ 天界 (tier=realm, mc=0, ch=5) [celestial]
    ├─ 天庭 (tier=continent, mc=29, ch=7)
```

**看什么：**
- 父子关系对不对？（花果山应在傲来国下，长安城不应在江州下）
- tier 对不对？（南赡部洲应是 continent 不是 site）
- 同一地方有没有重复出现？（南赡部洲/南膳部洲/南瞻部洲 是同一地名变体）

**这个文件不用改**，只是帮你快速发现问题。发现的问题在 `locations.json` 里修。

## 2. 人物 `xiyouji_characters.json`

### 数据格式

```json
{
  "name": "黄门官",
  "mention_count": 10,
  "first_chapter": 5,
  "last_chapter": 98,
  "is_valid_character": null,    // ← 你改这个
  "correct_name": null,          // ← 名字有误才填
  "_note": ""                    // ← 可选备注
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 真实人物，名字正确 | `"is_valid_character": true` |
| 真实人物，但名字不对 | `"is_valid_character": true, "correct_name": "正确名字"` |
| 不是人物（泛称/物品/职位） | `"is_valid_character": false, "_note": "原因"` |

### 示例

```json
{"name": "孙悟空", "is_valid_character": true}
```

```json
{"name": "黄门官", "is_valid_character": false, "_note": "泛称，指宫中传旨太监，非固定人物"}
```

```json
{"name": "白马", "is_valid_character": true, "correct_name": "白龙马"}
```

### 重点关注

- mc 高但看起来可疑的（泛称、通称、职务名）
- `is_valid_character` 只填 `true` 或 `false`
- `correct_name` 只有名字**有误**时才填（例如 "白马" → "白龙马"）

## 3. 别名 `xiyouji_aliases.json`

### 数据格式

```json
{
  "canonical_name": "孙悟空",
  "system_aliases": ["外公", "大圣", "大师兄", "大徒弟", "美猴王", "齐天大圣"],
  "mention_count": 36,
  "is_correct_grouping": null,   // ← 你改这个
  "wrong_aliases": [],            // ← 不属于此人的别名
  "missing_aliases": [],          // ← 遗漏的重要别名
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 所有别名都对 | `"is_correct_grouping": true` |
| 有不属于此人的 | `"is_correct_grouping": false, "wrong_aliases": ["外公"]` |
| 缺少重要别名 | `"missing_aliases": ["行者", "猴王"]` |
| 两者都有 | 同时填 wrong_aliases + missing_aliases |

### 示例

正确的组：
```json
{
  "canonical_name": "唐僧",
  "system_aliases": ["三藏", "唐三藏", "圣僧", "御弟"],
  "is_correct_grouping": true,
  "wrong_aliases": [],
  "missing_aliases": ["唐长老"]
}
```

有问题的组：
```json
{
  "canonical_name": "孙大圣",
  "system_aliases": ["哥哥", "外公", "大圣", "孙悟空", "小妖", "悟空"],
  "is_correct_grouping": false,
  "wrong_aliases": ["小妖", "哥哥", "外公"],
  "missing_aliases": ["齐天大圣"],
  "_note": "小妖是泛称；哥哥/外公是称谓不是别名"
}
```

### 重点关注

- 同一人物有多个 canonical（孙悟空/行者/孙大圣 各有一组 → 应该合并为一组）
- 别名里混入泛称（"哥哥"、"师父"、"外公" 不是别名）
- wrong_aliases 和 missing_aliases 都是**数组**格式 `["a", "b"]`

## 4. 地点 `xiyouji_locations.json`

### 数据格式

```json
{
  "name": "南赡部洲",
  "mention_count": 7,
  "tier": "site",
  "parent": "主世界",
  "layer": "overworld",
  "children_count": 1,
  "is_valid_location": null,    // ← 你改这个
  "correct_tier": null,         // ← tier有误才填
  "correct_parent": null,       // ← parent有误才填
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 地点正确，tier 和 parent 都对 | `"is_valid_location": true`（其他不用填） |
| 地点正确，tier 不对 | `"is_valid_location": true, "correct_tier": "continent"` |
| 地点正确，parent 不对 | `"is_valid_location": true, "correct_parent": "西牛贺洲"` |
| tier 和 parent 都不对 | 同时填 `correct_tier` 和 `correct_parent` |
| 不是地名（泛称/物品/描述） | `"is_valid_location": "DELETE", "_note": "原因"` |

### tier 速查表

| tier | 含义 | 西游记典型例子 |
|------|------|-------------|
| world | 最顶层 | 天下 |
| realm | 界域 | 天界、冥界 |
| continent | 大洲/大陆 | 东胜神洲、西牛贺洲、南赡部洲 |
| kingdom | 国 | 傲来国、天竺国、乌鸡国 |
| region | 山/海/大区域 | 花果山、灵山、东海 |
| city | 城/镇/州 | 长安城、车迟城 |
| site | 洞/寺/庄/宫 | 水帘洞、雷音寺、高老庄 |
| building | 殿/楼/房/亭 | 灵霄殿、金銮殿 |

### 示例

tier 错误：
```json
{"name": "南赡部洲", "is_valid_location": true, "correct_tier": "continent", "_note": "四大部洲之一"}
```

parent 错误：
```json
{"name": "长安城", "is_valid_location": true, "correct_parent": "南赡部洲", "_note": "唐朝都城，在南赡部洲"}
```

应该删除：
```json
{"name": "高山", "is_valid_location": "DELETE", "_note": "泛称，非专有地名"}
```

tier + parent 都错：
```json
{"name": "河南", "is_valid_location": true, "correct_tier": "region", "correct_parent": "南赡部洲"}
```

### 建议检查策略

- **mc ≥ 5**（约 50 个）：仔细看每一个
- **mc 1-4**（约 200 个）：快速扫，主要看有没有泛称混入
- **mc = 0**（约 500 个）：大部分是系统生成的归类节点，可以跳过，只看名字明显不对的

## 5. 关系 `xiyouji_relations.json`

### 数据格式

```json
{
  "person_a": "孙悟空",
  "person_b": "猪八戒",
  "system_type": "师兄弟",
  "mention_count": 45,
  "first_seen": "ch8",
  "correct_type": null,          // ← 你改这个
  "correct_category": null,      // ← 你改这个
  "_note": ""
}
```

### 怎么改

两个字段都要填：

| 字段 | 填什么 |
|------|--------|
| `correct_type` | 正确的关系类型名称（如"师兄弟"、"父子"、"主仆"、"敌对"） |
| `correct_category` | 六大类之一（见下表） |

如果 `system_type` 已经正确，`correct_type` 填和 `system_type` 一样的值即可。

### category 分类表

| category | 含义 | 包含的关系类型 |
|----------|------|-------------|
| `family` | 血缘/婚姻 | 父子、母子、兄弟、夫妻、祖孙 |
| `intimate` | 亲密非血缘 | 情侣、知己、结拜兄弟 |
| `hierarchical` | 上下级 | 师徒、君臣、上下级、主仆 |
| `social` | 社交 | 朋友、同门、邻居、相识 |
| `hostile` | 对立 | 敌对、仇敌、对手 |
| `other` | 其他 | 以上都不是 |

### 示例

system_type 正确：
```json
{
  "person_a": "孙悟空",
  "person_b": "唐僧",
  "system_type": "师徒",
  "correct_type": "师徒",
  "correct_category": "hierarchical"
}
```

system_type 不正确：
```json
{
  "person_a": "孙悟空",
  "person_b": "猪八戒",
  "system_type": "朋友",
  "correct_type": "师兄弟",
  "correct_category": "social",
  "_note": "同为唐僧徒弟，师兄弟关系"
}
```

关系复杂的情况（取主要关系）：
```json
{
  "person_a": "孙悟空",
  "person_b": "牛魔王",
  "system_type": "结拜兄弟",
  "correct_type": "结拜兄弟",
  "correct_category": "intimate",
  "_note": "曾结拜但后来敌对，取早期主要关系"
}
```

## 6. 空间关系（contains 方向） `xiyouji_spatial.json`

这组数据检查的是"A 包含 B"的方向是否正确。直接影响地图的层级结构。

### 数据格式

```json
{
  "chapter": 1,
  "source": "花果山",
  "target": "水帘洞",
  "relation": "contains",
  "confidence": "high",
  "is_direction_correct": null,   // ← 你改这个
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 方向正确（source 确实包含 target） | `"is_direction_correct": true` |
| 方向反了（target 应该包含 source） | `"is_direction_correct": false, "_note": "应反转"` |
| 不存在包含关系 | `"is_direction_correct": false, "_note": "非包含关系"` |

### 示例

```json
{"source": "花果山", "target": "水帘洞", "is_direction_correct": true}
```

```json
{"source": "水帘洞", "target": "花果山", "is_direction_correct": false, "_note": "应反转，花果山包含水帘洞"}
```

### 建议

- 618 条太多不用全看，**抽查 50-100 条**即可（文件按章节排序，可以抽几个章节看）
- 重点看 confidence=high 但方向可疑的

## 7. 章节抽检 `xiyouji_spot_check.json`

随机抽了 5 章，列出每章提取的人物、地点、事件。用于检查整体提取质量。

### 数据格式

```json
{
  "chapter": 15,
  "characters": ["孙悟空", "唐僧", "猪八戒"],
  "locations": ["高老庄", "浮屠山"],
  "events": ["悟空收服猪八戒", "三人继续西行"],
  "relationships_count": 12,
  "extraction_quality": null,   // ← 你改这个
  "_note": ""
}
```

### 怎么改

对照原文（或你的记忆），判断提取结果是否合理：

| 评级 | 含义 |
|------|------|
| `"good"` | 人物、地点、事件基本完整，无明显错误 |
| `"acceptable"` | 有少量遗漏或错误，但整体可用 |
| `"poor"` | 重大遗漏或大量错误 |

### 示例

```json
{
  "chapter": 15,
  "extraction_quality": "good",
  "_note": "主要人物和地点都提取到了"
}
```

```json
{
  "chapter": 8,
  "extraction_quality": "acceptable",
  "_note": "提取了44个人物偏多，可能包含泛称角色"
}
```

---

## 通用注意事项

1. **只改需要改的字段**，正确的保持 `null` 或不动
2. **`_note` 可选**，遇到不确定的或想说明理由时写
3. **数组字段**（wrong_aliases、missing_aliases）用 `["a", "b"]` 格式
4. **地点检查建议从高频开始**，低频的快速扫即可
5. 检查完后文件放回 `backend/data/review/` 原位
