> 基于 v0.70 重新分析的数据，含 NameResolver（人名统一）、泛称过滤升级、地点知识补充。

## 文件清单

每本小说 7 个文件，前缀分别为 `xiyouji_` 和 `honglou_`：

**两本都检查完预计 3-4 小时。** 如果时间紧，优先检查西游记（与上轮对比验证修复效果）。

## 1. 层级树 `{key}_hierarchy_tree.txt`

**怎么看：** 直接打开，看缩进结构是否合理。

**看什么：**
- 天下只有 3-5 个子节点（主世界/天界/冥界/龙宫）？
- 四大部洲都是 continent？
- 天庭是 region（不是 continent）？
- 父子关系对不对？（长安城在东土大唐/南赡部洲下，不在江州下）
- 同一地名有没有重复？（南赡部洲/南膳部洲 应已合并）

**这个文件不用改**，发现的问题在 `locations.json` 里修。

## 2. 人物 `{key}_characters.json`

### 数据格式

```json
{
  "name": "黄门官",
  "mention_count": 10,
  "first_chapter": 5,
  "last_chapter": 98,
  "is_valid_character": null,
  "correct_name": null,
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 真实人物，名字正确 | `"is_valid_character": true` |
| 真实人物，但名字不对 | `"is_valid_character": true, "correct_name": "正确名字"` |
| 不是人物（泛称/物品/职位） | `"is_valid_character": false, "_note": "原因"` |

### 本轮重点

- **v0.70 新增了 NameResolver**——检查"行者"和"孙悟空"是否已合并为一个（不再分开出现）
- **泛称过滤升级**——检查"黄门官""文武多官"等是否已被过滤掉
- 如果上轮标记为无效的条目仍然出现，说明过滤没生效，请标注并注明

## 3. 别名 `{key}_aliases.json`

### 数据格式

```json
{
  "canonical_name": "孙悟空",
  "system_aliases": ["大圣", "美猴王", "齐天大圣"],
  "mention_count": 36,
  "is_correct_grouping": null,
  "wrong_aliases": [],
  "missing_aliases": [],
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 所有别名都对 | `"is_correct_grouping": true` |
| 有不属于此人的 | `"is_correct_grouping": false, "wrong_aliases": ["外公"]` |
| 缺少重要别名 | `"missing_aliases": ["行者", "猴王"]` |
| 两者都有 | 同时填 wrong_aliases + missing_aliases |

### 本轮重点

- **泛称是否还混入？**（哥哥/师父/外公/徒弟 应已被过滤）
- **同一人物是否还有多组？**（行者/孙悟空/孙大圣 应已合并为一组）
- **叙述片段是否还出现？**（"大圣曾养过马" 应已被过滤）

## 4. 地点 `{key}_locations.json`

### 数据格式

```json
{
  "name": "南赡部洲",
  "mention_count": 7,
  "tier": "continent",
  "parent": "主世界",
  "layer": "overworld",
  "children_count": 5,
  "is_valid_location": null,
  "correct_tier": null,
  "correct_parent": null,
  "_note": ""
}
```

### 怎么改

| 情况 | 操作 |
|------|------|
| 正确 | `"is_valid_location": true` |
| tier 不对 | `"is_valid_location": true, "correct_tier": "region"` |
| parent 不对 | `"is_valid_location": true, "correct_parent": "西牛贺洲"` |
| 不是地名 | `"is_valid_location": "DELETE", "_note": "原因"` |

### tier 速查表

| tier | 含义 | 西游记典型 | 红楼梦典型 |
|------|------|----------|----------|
| world | 最顶层 | 天下 | 天下 |
| realm | 界域 | 主世界、天界、冥界 | 主世界 |
| continent | 大洲 | 东胜神洲、西牛贺洲 | — |
| kingdom | 国 | 傲来国、天竺国 | — |
| region | 山/海/大区 | 花果山、灵山 | 金陵、荣国府 |
| city | 城/镇/州/府 | 长安城、车迟城 | 都中、扬州 |
| site | 洞/寺/庄/宫 | 水帘洞、高老庄 | 大观园、宁国府 |
| building | 殿/楼/房/亭 | 灵霄殿、金銮殿 | 潇湘馆、怡红院 |

### 本轮重点

- **泛称地点是否减少？**（方丈/禅堂/高山/山凹 应已被过滤）
- **tier 修正是否生效？**（天庭→region, 南赡部洲→continent）
- **parent 修正是否生效？**（长安城→东土大唐, 大雷音寺→灵山）
- **地名变体是否合并？**（南膳部洲→南赡部洲）

### 检查策略

- **mc ≥ 5**（约 50 个）：仔细看每一个
- **mc 1-4**（约 200 个）：快速扫，主要看泛称
- **mc = 0**（约 500 个）：大部分是系统节点，只看名字明显不对的

## 5. 关系 `{key}_relations.json`

### 数据格式

```json
{
  "person_a": "孙悟空",
  "person_b": "猪八戒",
  "system_type": "师兄弟",
  "mention_count": 45,
  "first_seen": "ch8",
  "correct_type": null,
  "correct_category": null,
  "_note": ""
}
```

### 怎么改

两个字段都要填：
- `correct_type`：正确的关系类型（如 system_type 对的就填一样的）
- `correct_category`：`family` / `intimate` / `hierarchical` / `social` / `hostile` / `other`

### category 分类表

| category | 含义 | 包含的关系 |
|----------|------|-----------|
| `family` | 血缘/婚姻 | 父子、母子、兄弟、夫妻、祖孙 |
| `intimate` | 亲密非血缘 | 情侣、知己、结拜兄弟 |
| `hierarchical` | 上下级 | 师徒、君臣、上下级、主仆 |
| `social` | 社交 | 朋友、同门、师兄弟、邻居、相识 |
| `hostile` | 对立 | 敌对、仇敌、对手 |
| `other` | 其他 | 以上都不是 |

### 本轮重点

- **碎片关系是否消除？**（上轮唐僧↔孙悟空有 12 条碎片，本轮应只有 1-2 条）
- **如果还有碎片**，说明 NameResolver 没完全生效，请标注
- 关系类型判断和上轮一样

```json
  "target": "水帘洞",
  "relation": "contains",
  "confidence": "high",
  "is_direction_correct": null,

```

### 怎么改

| 情况 | 操作 |
|------|------|
| 方向正确 | `"is_direction_correct": true` |
| 方向反了 | `"is_direction_correct": false, "_note": "应反转"` |
| 非包含关系 | `"is_direction_correct": false, "_note": "非包含关系"` |

**抽查 50-100 条即可**，不用全看。

## 7. 章节抽检 `{key}_spot_check.json`

### 数据格式

```json
{
  "chapter": 15,
  "characters": ["唐僧", "孙悟空", "猪八戒"],
  "locations": ["高老庄", "浮屠山"],
  "events": ["悟空收服猪八戒", "三人继续西行"],
  "relationships_count": 12,
  "extraction_quality": null,
  "_note": ""
}
```

### 怎么改

`extraction_quality`：`"good"` / `"acceptable"` / `"poor"`

### 本轮重点

- **events 字段应该有内容了**（上轮因导出 bug 全空，已修复）
- **人物名是否已统一？**（应该看到"孙悟空"而非"行者"+"孙悟空"混用）
- 如果某章仍然 events 全空，说明该章提取有问题，请标注

## 通用注意事项

1. **只改需要改的字段**，正确的保持 `null` 或不动
2. **`_note` 可选**，遇到不确定的或想说明理由时写
3. **数组字段**（wrong_aliases、missing_aliases）用 `["a", "b"]` 格式
4. **地点从高频开始看**，低频快速扫即可
5. 检查完后文件加 `_reviewed` 后缀保存（如 `xiyouji_characters_reviewed.json`）
6. **写一份总结报告**（如 `xiyouji_review_summary_v2.md`），对比上轮指出改善和新发现的问题
