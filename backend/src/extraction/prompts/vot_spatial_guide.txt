## 空间推理引导（Scene Graph CoT — 请在提取 spatial_relationships 前先完成以下结构化推理）

在提取空间关系之前，请先构建本章的**空间场景图**，然后从场景图中派生 spatial_relationships。

**第一步：地名清单与层级**
回顾本章文本，按层级列出所有出现的地名：
- 宏观（大洲/大区域/国/界）→ 中观（城/山/域）→ 微观（建筑/洞府/房间）
- 标注每个地名的"容器"：哪个更大的地点包含它？

**第二步：场景图构建（核心步骤）**
在脑海中构建一张有向图：
- 节点 = 地点，带属性（规模层级、类型）
- 边 = 空间关系（方位/包含/相邻/路径）
- ⚠️ **contains 方向校验（逐条检查）**：
  - 对每个 contains，用后缀判断层级：洲/洋 > 国/府(行政) > 山/海/江/湖/县 > 城/镇 > 村/洞/寺/宫/园 > 殿/楼/房/院/台/门
  - ⚠️ "府"有两义：行政府（开封府≈州级=2）vs 宅邸（荣国府/王府≈院落级=5）
  - source 的后缀层级必须 ≤ target（source 更大或同级）
  - ✅ 花果山(山=3) contains 水帘洞(洞=5) → 3<5 正确
  - ✅ 金陵城(城=4) contains 荣国府(国府=宅邸=5) → 4<5 正确
  - ✅ 龙宫(宫=5) contains 水晶殿(殿=6) → 5<6 正确
  - ❌ 水帘洞(洞=5) contains 花果山(山=3) → 5>3 反转！交换
  - ❌ 梨香院(院=6) contains 荣国府(府=2) → 6>2 反转！交换
  - 如果无法从后缀判断，看名字长度：更长的名字通常是更小的地点

**第三步：移动路径追踪**
如果本章有角色在不同地点间移动：
- 完整记录移动路线：出发地 → 途经地 → 目的地
- 使用 travel_path 关系记录，waypoints 字段列出中间经过的地点
- 移动暗示的空间关系：先到A再到B → A和B相邻或A在出发地和B之间

**第四步：距离与规模判断**
关注文中对距离和规模的描述：
- 明确距离词："三天路程"→distance_class=far，"近在咫尺"→near
- 规模对比线索："小镇"→city/site，"方圆千里"→kingdom/continent

**第五步：自检与输出**
基于场景图提取 spatial_relationships 时，逐条验证：
1. source 和 target 都在本章地名清单中吗？
2. 如果是 contains，source 的层级 > target 的层级吗？
3. confidence_score 有文本依据支撑吗？（直接描述=0.9，暗示推理=0.7，模糊推测=0.5）
4. 目标：每章产出 4-8 条空间关系
