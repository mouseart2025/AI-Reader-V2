> 生成日期：2026-02-15
> 基于：Report A（全景战略）+ Report B（战术方案）+ 2026 年最新市场数据

## 一、2026 年市场验证

### 1.1 竞品最新动态

**创作者工具阵营（直接竞品）**：

- **Plottr**（[2026 路线图](https://plottr.com/plottr-in-2026/)）：2025 年推出了 Family Tree 和 Idea Threads 功能，但明确表态「反 AI」，强调 human-centered storytelling。这是机会窗口——Plottr 主动放弃了 AI 赛道。
- **NovelCrafter**（[评测](https://intellectualead.com/novelcrafter-review-guide/)）：迅速崛起为「AI 写作的 Photoshop」，BYOK 模式支持 300+ 模型。Codex（Story Bible）是其核心卖点，但它是**手动录入 + AI 辅助生成**，不是从已有文本自动提取。定价 $4-$20/月。
- **Sudowrite**（[评测](https://nerdynav.com/sudowrite-review/)）：Story Bible 功能维护一致性，Muse 模型专为小说微调。定价 $10-$44/月。但是云端服务，数据在平台侧。
- **World Anvil / Campfire**：仍然是手动录入为主的世界构建工具，没有自动提取能力。

**关键发现**：NovelCrafter 和 Sudowrite 正在证明「AI + Story Bible」的市场需求，但它们都是**正向创作工具**（写→生成），没有做**逆向分析工具**（已有文本→自动提取）。AI Reader V2 的「自动化逆向工程」是独占生态位。

**大厂 AI 写作工具**：

- **番茄小说**：2024 年上线 AI 辅助写作，但 2026 年 2 月启动「专项整治打击低质量 AI 量产内容」。同时 AI 训练协议争议持续，7000 万用户的网文平台打出了[「反 AI」第一枪](https://news.qq.com/rain/a/20250307A04RYC00)
- **阅文妙笔**：2025 年接入 DeepSeek-R1，AI 功能周使用率达 50%，但完全是云端服务，数据归平台。作者信任危机依然存在
- **行业数据**：[中国 AI 写作工具市场](https://blog.csdn.net/yao1500/article/details/150617841) 2025 年达 1280 亿元，同比增长 75%。但 80%+ 小说作者反映「设定混乱」严重影响创作

**结论**：「面向长篇作者/编辑的本地 Series Bible 生成器」定位**依然成立且更加强化**：
1. 大厂的隐私争议越来越严重，本地化价值持续上升
2. NovelCrafter/Sudowrite 验证了 BYOK + Story Bible 的付费意愿，但没有做逆向提取
3. Plottr 主动放弃 AI，留出了巨大空白
4. 中文市场缺乏同类竞品（橙瓜码字、壹写作等功能远不及此）

### 1.2 BYOK 模式的市场验证

BYOK 已成为 2025-2026 年的主流趋势：
- [JetBrains AI](https://blog.jetbrains.com/ai/2025/12/bring-your-own-key-byok-is-now-live-in-jetbrains-ides/) 2025 年底正式上线 BYOK
- [NovelCrafter](https://www.novelcrafter.com/) 以 BYOK 为核心卖点
- [BYOKList.com](https://byoklist.com/) 专门收录 BYOK 工具

用户已经接受「软件授权 + 自带 Key」的付费模式。

## 二、差距分析：当前功能 vs 付费所需功能

| 付费功能 | 当前状态 | 差距 | 工作量 |
|---------|---------|------|--------|
| **Series Bible 导出 (PDF/MD/Docx/JSON)** | 已有 JSON 导出 | 缺 PDF/Docx 排版导出、缺「设定集模板」 | 中 |
| **一致性/穿帮检测** | 无 | 需要全新模块：比对 entity profile 跨章节属性矛盾 | 大 |
| **设定锁定/版本化** | 有 override 机制（map_user_overrides, world_structure_overrides） | 需要扩展到所有实体类型，加 UI 锁定标记 | 中 |
| **无剧透问答（按进度）** | RAG 问答已有，chapter_start/end 过滤已有 | 需要 UI 包装为「写作进度模式」 | 小 |
| **类型模板/提示词包** | extraction_system.txt 是通用 prompt | 需要仙侠/玄幻/悬疑/历史等分类 prompt 包 | 小 |
| **授权/License 系统** | 无 | 需要 license key 验证 + 功能门控 | 中 |
| **桌面打包 (Electron/Tauri)** | 纯 Web（需手动启动前后端） | 需要打包为一键启动的桌面应用 | 大 |
| **落地页 + 支付** | 无 | 需要产品官网 + 支付集成 | 中 |

**核心已有资产**（不需要改的）：
- 章节分割 + 实体预扫描 ✅
- 逐章 LLM 提取 + 别名合并 ✅（已完成分级安全优化）
- 人物卡片（含关系分类、多证据） ✅（已完成改进）
- 关系图谱（力导向图、按类型着色） ✅（已完成改进）
- 多层世界地图 ✅
- 事件时间线 ✅
- 百科设定集 ✅
- RAG 问答 ✅
- 本地 Ollama + OpenAI 兼容 API ✅

## 三、系统性执行计划

### Phase 0：MVP 验证（最小可行付费版）

**目标**：用最小成本验证「作者愿意为自动 Series Bible 付费」

**关键任务**：

1. **Series Bible 导出 — Markdown 版**
   - 将现有的 PersonProfile / LocationProfile / 事件时间线 聚合为一个格式化的 Markdown 文档
   - 结构：封面 → 目录 → 人物小传（含关系分类）→ 地点百科 → 势力组织 → 物品设定 → 时间线 → 关系图（静态截图）
   - 这是最快能实现的导出形式，后续再做 PDF/Docx

2. **落地页 + Demo 视频**
   - 一页式官网：定位语 + 3 分钟 Demo 视频（TXT 导入 → 自动生成地图/关系图/人物卡/设定集 → 一键导出）
   - 早鸟价预售表单（不需要支付系统，先收邮箱/微信）

3. **目标用户访谈（20 人）**
   - 从起点/番茄/晋江找 10 个百万字级连载作者
   - 从编辑/校对圈找 5 人
   - 从同人/设定党找 5 人
   - 核心问题：现在怎么管理人物/设定/时间线？痛点是什么？愿意付多少？

**里程碑**：收到 20+ 早鸟意向注册，5+ 用户明确表示愿意付费

---

### Phase 1：核心付费功能开发

**目标**：完成 Pro 版的 5 个付费钩子

**按优先级排序**：

**P0 — Series Bible 导出增强**
- PDF 导出（使用 WeasyPrint 或 reportlab）
- Docx 导出（使用 python-docx）
- 可选模板（简洁版 / 详细版 / 编辑投稿版）
- 这是转化率最高的功能：「把你的草稿变成资产」

**P0 — License 系统 + 功能门控**
- 简单的 license key 验证（本地校验，无需联网）
- Free 版限制：1 本书 / 最多 50 章 / 无导出
- Pro 版：无限制 + 导出 + 后续所有 Pro 功能

**P1 — 无剧透写作模式**
- UI 开关：「写作进度锁定到第 N 章」
- 问答/百科/图谱自动过滤到 N 章范围
- 已有 chapter_start/end 基础设施，只需前端包装

**P1 — 设定锁定/版本化**
- 将现有 override 机制扩展到人物/关系/组织
- UI：在人物卡片上加「锁定」按钮
- 锁定的实体在重分析时不会被覆盖

**P2 — 类型模板**
- 仙侠/玄幻模板：修仙境界体系、灵根属性、宗门结构
- 悬疑模板：线索链、嫌疑人、时间线冲突
- 历史模板：朝代背景、官职体系
- 实现：不同的 extraction_system.txt prompt 变体

**P2 — 一致性检测（基础版）**
- 阶段一：属性矛盾检测（同一人物在不同章的矛盾属性）
- 阶段二：时空矛盾检测（地理移动不合理）
- 这是最有技术门槛的功能，但也是最强的付费点

**里程碑**：Pro 版可以开始销售

---

### Phase 2：市场推广与早期销售

**目标**：获得前 100 个付费用户

**关键任务**：

1. **桌面打包**
   - 使用 Tauri（Rust + WebView）打包为 macOS / Windows 一键安装包
   - 内嵌 Python 后端 + 前端
   - 这是降低用户使用门槛的关键步骤

2. **定价与支付**
   - Pro（个人作者）：¥299 买断 / 含 1 年更新，续费 ¥99/年
   - Studio（工作室）：¥999/5 席位/年
   - 国际价格：$49 / $99
   - 支付：国内用微信/支付宝（可用 LemonSqueezy 或手动转账），国际用 Gumroad/LemonSqueezy

3. **推广渠道**
   - **内容营销**：在起点/番茄作者论坛、龙空、知乎「网文写作」话题发教程
   - **Demo 视频**：B 站/YouTube 发「我用 AI 自动整理了我的 300 万字小说设定」
   - **种子用户**：找 5-10 个有一定粉丝量的网文作者做内测，给免费 License 换 review
   - **Product Hunt / V2EX / 少数派**：面向技术+创作交叉人群

4. **反馈循环**
   - 内测群（微信/Discord）收集需求
   - 每 2 周迭代一个版本

**里程碑**：100 个付费用户，月收入 ¥5000+，NPS > 40

---

### Phase 3：产品成熟与扩展

**目标**：扩大用户基数，探索 B2B

**关键任务**：

1. **一致性检测增强版（Logic Guardian）**
   - 完整的「穿帮报告」：人物年龄矛盾、称呼变化、功法设定冲突、地理时空冲突
   - 这是 Report A 中提到的「IP 连续性审计引擎」的简化版
   - 目标：成为核心差异化卖点

2. **多语言支持**
   - 英文/日文小说支持
   - 打开国际市场（NovelCrafter 的用户群是英文为主）

3. **VTT 导出（探索性）**
   - Foundry VTT module.json 导出
   - 目标用户：跑团 GM

4. **B2B 探索**
   - 影视/动漫工作室的 IP 审计需求
   - 出版社编辑的校对辅助
   - 按私有化部署收费

**里程碑**：1000+ 付费用户，年收入 ¥30 万+，开始探索企业客户

## 四、风险与应对

| 风险 | 应对 |
|------|------|
| LLM 提取准确率不够高 | 溯源引用 + 置信度标记 + 人工修正锁定（已有 override） |
| 大厂推出类似功能 | 大厂做的一定是云端，隐私是我们的护城河 |
| BYOK 门槛高（普通作者不会配置 API） | 提供 Ollama 一键安装引导 + 预配置的便宜 API（如 DeepSeek） |
| 付费意愿不足 | Phase 0 先验证，不验证通过不投入 Phase 1 |
| 桌面打包技术复杂度 | Tauri 生态成熟，优先支持 macOS（目标用户设备） |

## 五、总结

**核心判断**：2026 年的市场环境比 Report A/B 撰写时更有利：
1. NovelCrafter/Sudowrite 用真实收入验证了 BYOK + Story Bible 的付费意愿
2. 大厂隐私争议加剧（番茄整治、阅文协议），本地化价值持续上升
3. Plottr 主动放弃 AI 赛道，留出生态位
4. 中文 AI 写作市场 1280 亿但缺乏专业逆向分析工具

**第一步行动**：实现 Series Bible Markdown 导出 + 落地页 + 20 人访谈。这三件事可以并行推进，是成本最低的验证方式。
